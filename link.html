Etiquetas Link


¿Qué es la etiqueta <link> en HTML?

La etiqueta HTML <link> es una etiqueta de encabezado que se utiliza para establecer una relación entre un documento HTML y un recurso externo, como una hoja de estilos CSS, un icono de favicon o un archivo de fuente. La etiqueta <link> se utiliza para definir el tipo de relación que existe entre el documento HTML y el recurso externo. Por ejemplo, la etiqueta <link> se utiliza para establecer la relación entre un documento HTML y una hoja de estilos CSS, y se utiliza el atributo “rel” para establecer la relación. Si el atributo “rel” se establece en “stylesheet”, se indica que el recurso externo es una hoja de estilos. La etiqueta <link> es una de las etiquetas de encabezado más importantes de HTML5 y se utiliza ampliamente en la construcción de sitios web modernos. Además de vincular hojas de estilos y otros recursos, la etiqueta <link> también se utiliza para establecer la relación entre documentos HTML y diferentes versiones del mismo documento (como una versión en un idioma diferente) o para indicar la relación entre documentos HTML y feeds RSS o Atom. Es importante tener en cuenta que la etiqueta <link> debe estar ubicada en la sección <head> del documento HTML y debe estar cerrada con una barra diagonal, como sigue: <link rel="stylesheet" href="styles.css" />.>



Atributos

Aquí te muestro una lista de los atributos más comunes de la etiqueta de HTML5:


  • href: especifica la ubicación del recurso vinculado, que puede ser una URL, una ruta relativa o un archivo local.
  • rel: especifica el tipo de relación entre el documento actual y el recurso vinculado. Algunos de los valores más comunes para “rel” son:
    1. “stylesheet”: se utiliza para vincular una hoja de estilo CSS al documento actual.
    2. “icon”: se utiliza para especificar un archivo de icono para el documento actual, como un favicon.
    3. “preload”: se utiliza para cargar un recurso de forma previa para mejorar el rendimiento de la página.
    4. “preconnect”: se utiliza para establecer una conexión previa con un servidor externo para mejorar el rendimiento de la página.
    5. “dns-prefetch”: se utiliza para realizar la resolución de DNS de un dominio de forma previa para mejorar el rendimiento de la página.
    6. “canonical”: se utiliza para especificar la URL canónica del documento actual, es decir, la URL preferida para acceder al contenido de la página.
    7. “alternate”: se utiliza para especificar versiones alternativas del documento actual, como versiones en otros idiomas o formatos.
    8. “next” y “prev”: se utilizan para indicar la relación de páginas anteriores y siguientes en una serie de documentos.
    9. “author”, “publisher” y “me”: se utilizan para especificar información sobre el autor, el editor y el propietario de la página, respectivamente.
    10. “search”: se utiliza para especificar un formulario de búsqueda para la página actual.
  • type: especifica el tipo MIME del recurso vinculado. Por ejemplo, para hojas de estilo CSS, el tipo MIME es “text/css”, mientras que para archivos de iconos, el tipo MIME puede ser “image/png” o “image/x-icon”.
  • sizes: se utiliza junto con “rel” igual a “icon” para especificar los tamaños del icono vinculado. Puede tomar valores en píxeles o en porcentaje.
  • media: especifica las condiciones de presentación del recurso vinculado, como la resolución de pantalla, el ancho de la ventana, etc. Se utiliza principalmente para hojas de estilo CSS, y puede tomar valores como “all”, “screen”, “print”, entre otros.
  • integrity: permite especificar una firma criptográfica que permite verificar la integridad del recurso vinculado. Se utiliza principalmente para asegurar la autenticidad y la integridad de los recursos de terceros.
  • crossorigin: se utiliza para especificar si el recurso vinculado puede ser solicitado desde una fuente distinta al dominio que sirve el archivo HTML. Puede tomar valores como “anonymous” o “use-credentials”.
